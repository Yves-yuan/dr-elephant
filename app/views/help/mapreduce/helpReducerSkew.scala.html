This analysis shows whether there is a data/time-skew for the data/time entering reducer tasks.

This result of the analysis shows two groups of the spectrum, where the first group has significantly less input data compared to the second group.

Example

Reducer Skew

Severity: Critical
Number of tasks 999
Group A 875 tasks @@ 28 MB avg
Group B 124 tasks @@ 2 GB avg

Suggestions

This is often caused by skew in the keyspace (aggregation key for group by, join key for joins).
If using Pig, try a skew join.
Otherwise, consider what you're doing in the job and if there's a better way to do it.
